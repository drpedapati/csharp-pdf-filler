# Support

Use GitHub issues for:
- reproducible bugs
- feature requests
- release and packaging problems
- documentation gaps

Before opening an issue:
- run the latest release or current `main`
- include the exact command and relevant output
- use sanitized or public PDFs only

Do not attach:
- real patient or member PDFs
- regulated claims or authorization documents
- secrets, credentials, or license keys

For security-sensitive issues, use [SECURITY.md](SECURITY.md) instead of a public issue.
